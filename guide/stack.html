サービス一覧と代表ポート

Git や Docker に不慣れな方でも混乱しないよう表形式でまとめました。

サービス 役割と保存先 ポート / 備考
mmam FastAPI。RESTエンドポイント／Scheduler／NMOSツールがここに集約。 8080 (HTTP) / 8443 (HTTPS)。TLS証明書は certs/。
ui Vue 3 UI。frontend/dist を nginx が配信。 4173 / 4174。UIがAPI URLを自動判別。
db PostgreSQL 16。flows, planner_buckets, automation_history など。 5432 (内部)。永続化は ./pgdata。
mqtt Mosquitto。REST更新を MQTT/MQTT-WS でブロードキャスト。 1883 / 9001。無効化したい場合は MQTT_ENABLED=false。

FastAPIレイヤー

8つのルーターをモジュール化し、APScheduler がバックグラウンドジョブを統括します。

  • routers/flows.py にREST PATCHとChecker APIを集約
  • NMOS連携は routers/nmos.py と services/nmos
  • ログは /var/log/mmam に日次でローテート

PostgreSQL

schema.sql で初期化されるテーブル構造。テキストハブ用フィールドはUTF-8を前提にしています。

  • flows.alias_1-8 / user_field_1-8 がメモ領域
  • planner_buckets は再帰CTEで階層を取得
  • バックアップは docker exec db pg_dump mmam > backup.sql

通信フロー（文章による図解）

  1. ブラウザは https://localhost:4174（または4173）へアクセス。UIがAPIベースURLを自動計算。
  2. ユーザー操作 → UIがREST API(8443)へPOST/PATCH。TLS証明書は certs/ で差し替え可能。
  3. FastAPIはPostgreSQLへSQLを投げ、結果をレスポンスとして返す。同時に変更イベントをMQTTへpublish。
  4. CheckerやNMOSジョブはFastAPI内部でスケジュールされ、結果をDB + MQTTに書き戻す。

ゼロトラスト環境では UI→API を社内CAで暗号化し、API→DB は Docker ネットワーク内に閉じていることを確認してください。
