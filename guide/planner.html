Planner / Address Map 完全ガイド

Excelで管理していたマルチキャストアドレス帳を安全に移行し、常に最新の予約状況を共有できます。Landingへ戻るボタンを常設。

Landingへ戻る Address API

UI操作の基本

  1. Drive (/8) を作成し、「Production」「DR」などのタグを付与。
  2. Folder (parent) を /16〜/20単位で追加し、色や説明を設定。
  3. View (child) を最大4,096アドレスで作成。Explorerでブロックを視覚化。
  4. セルをクリックして予約メモ・担当者・期限を入力。Flowと紐付ける場合は flow_id を記載。

Explorer では /24 境界で罫線が強調されるため、ネットワーク初心者でもブロックサイズを直感的に把握できます。

Excelからの移行

  • 既存台帳を Drive, Folder, CIDR, 備考 の列に整形してCSV保存。
  • サンプルスクリプト scripts/import-planner.py を使い、APIへ順次POST。
  • GET /address/buckets/export でJSONバックアップを取得し、Git管理を推奨。

APIチートシート

# 階層一覧
curl https://localhost:8443/address/buckets/overview

# 子ビュー作成
curl -X POST https://localhost:8443/address/buckets/child \
  -H \"Content-Type: application/json\" \
  -d '{\"parent_id\": 4, \"name\": \"Studio-A/239.10.0.0/22\", \"cidr\": \"239.10.0.0/22\" }'

# Explorer用マップ
curl https://localhost:8443/address-map?view_id=12

運用Tips

  • 予約メモに #タグ を入れると検索しやすくなります。
  • Flow作成時は Planner からコピーできるため、アドレス重複が激減します。
  • 月初の自動バックアップを Automation で設定すると監査が簡単です。
