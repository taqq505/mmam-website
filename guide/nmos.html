UIワークフロー（Wizard）

  1. Discover — ベースURLとAPIバージョン(IS-04/05)を入力し、ノードを探索。
  2. Select Devices — Flowと紐付けたいSender/Receiverをチェック。既存Flowがあれば自動マッピング。
  3. Diff — IS-05のtransport_paramsとMMAM側のFlow定義を比較し、差分を色付きで表示。
  4. Apply — 必要に応じて個別/一括適用。IS-05 PATCHが実行され、結果はAuditログに残ります。

Wizardのどのステップでもスクリーンショットを取得しやすいようUIが段階表示されています。

主要API

  • POST /nmos/discover — ベースURLを入力するとNode/Sender情報を収集。
  • GET /flows/{id}/nmos/check — Flow vs NMOSの差分をJSONで取得。
  • POST /flows/{id}/nmos/apply — 差分をIS-05へ反映。
curl -X POST https://localhost:8443/nmos/discover \ 
  -H \"Content-Type: application/json\" \
  -d '{\"base_url\": \"http://node01:3205\", \"version\": \"v1.3\" }'

トラブルシューティング

  • SSLが必要なNMOSシステムは NMOS_TLS_VERIFY=false で検証を一時的に無効化。
  • 差分が大きすぎる場合は flows/{id}/export でJSONを出力し、手動で確認。
  • Apply失敗時はHTTPレスポンスにデバイス側のエラーが含まれるため、ログを参照。
