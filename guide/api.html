認証と基本リクエスト

  1. POST /login にユーザー名/パスワードを送信し、JWTを取得。
  2. 以降のRESTリクエストでは Authorization: Bearer <token> を付与。
  3. トークンは1時間有効。自動スクリプトは期限切れ前にリフレッシュするか再ログイン。
curl -X POST https://localhost:8443/login \\
  -H \"Content-Type: application/json\" \\
  -d '{\"username\":\"admin\",\"password\":\"admin\"}'

主要RESTエンドポイント

  • GET /flows, POST /flows, PATCH /flows/{id}
  • GET /address/buckets/overview, POST /address/buckets/child
  • GET /checker/latest, POST /automation/jobs/{id}/enable
  • GET /logs?kind=api|audit

MQTTトピック

  • mmam/flows/events — Flow作成/更新/削除。
  • mmam/checker/events — 自動検証の結果。
  • mmam/automation/jobs/<id> — ジョブ状態の推移。
// MQTT.js例
client.subscribe('mmam/flows/events');
client.on('message', (_, payload) => {
  const event = JSON.parse(payload.toString());
  console.log(event.flow_id, event.event);
});

システム連携Tips

  • REST PATCH を利用すると既存Flowを壊さず差分更新できます。外部システムのキーと flow_id を紐付けておくと便利。
  • Automation結果をIssue Trackerに上げたい場合は、MQTTイベントをWebhookへ転送するだけでOK。
  • Flow Export/Import APIを使えば環境間の同期が容易。CIで GET /flows/export を取得→Gitに保存する運用も可能です。
