衝突検出とNMOS差分チェックを24/7で回すための設定手順とAPIを紹介します。Landingへのショートカットもこちらに。

UIでの進め方

  1. Checkerタブで「Collision」「NMOS」の Run Now を実行して現状を確認。
  2. 「Enable Automation」を押し、Interval（例: 10分）または Cron（例: 0 * * * *）を設定。
  3. RESTだけで運用したい場合は MQTT_ENABLED=false にし、外部システムが /checker/latest をポーリング。

結果は Automation history と Audit ログに記録され、誰が設定したか追跡できます。

Checker API

  • GET /checker/collisions — 重複S/G/Portをリスト表示。
  • GET /checker/nmos?timeout=5 — NMOS差分を取得。
  • GET /checker/latest?kind=collisions — 直近実行の要約。
curl https://localhost:8443/checker/latest?kind=collisions \\
  -H \"Authorization: Bearer <token>\"

Automation API

  • GET /automation/jobs — すべてのジョブを取得。
  • PUT /automation/jobs/{job_id} — Interval/Cron設定を更新。
  • POST /automation/jobs/{job_id}/enable — ジョブを有効化/無効化。
curl -X PUT https://localhost:8443/automation/jobs/collision \\
  -H \"Content-Type: application/json\" \\
  -d '{\"schedule_type\":\"cron\",\"cron\":\"0 * * * *\"}'
