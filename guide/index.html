learning path

推奨の読み進め順

  1. Setup — 開発環境の前提知識と docker compose の初回起動。
  2. Stack — FastAPI/DB/UIの関係を押さえ、どの設定がどこに保存されるかを理解。
  3. Flows / Planner — 実際のアドレス計画とテキスト共有を UI と API の両面で把握。
  4. NMOS / Checker — デバイス登録や自動検証を用いて、運用の再現性を確保。
  5. API — 既存システムや社内スクリプトから呼び出すための詳細リファレンス。

気になるページだけを参照しても構いません。各ページ末尾には関連ページへの導線や広告枠を用意しています。
