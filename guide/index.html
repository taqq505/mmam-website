learning path

読んでいただく際のガイドライン

  1. Setup — 開発環境の前提知識と Docker Compose による初回起動方法を解説します。
  2. Stack — FastAPI / DB / UI の関係を押さえ、どの設定がどこに保存されるかを確認します。
  3. Flows / Planner — 実際のアドレス計画とテキスト共有を UI と API の両面で把握します。
  4. NMOS / Checker — デバイス登録や自動検証を用い、運用の再現性を確保します。
  5. API — 既存システムや社内スクリプトから呼び出すための詳細リファレンスです。

気になるページだけを参照いただいても問題ありません。各ページ末尾には関連ページへの導線や広告枠を用意しています。

このガイドで得られること

  • ST2110向けのマルチキャストアドレス設計ノウハウ
  • REST API / MQTT / Scheduler をまたいだ実装例
  • 障害調査時に頼れるテキストハブ化のポイント
