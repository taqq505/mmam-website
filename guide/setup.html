ステップバイステップ

  1. リポジトリを取得 — git clone https://github.com/taqq505/mmam-docker.git
  2. .envを作成 — cp .env.example .env 後、POSTGRES_PASSWORD や SECRET_KEY を任意値に変更。
  3. Dockerを起動 — docker compose up --build。初回は依存イメージの取得で数分かかります。
  4. ブラウザで確認 — http://localhost:4173 または https://localhost:4174 にアクセスし、admin / admin でログイン。
  5. 初期データ投入 — Flow/Planner/NMOSのテストデータを登録して操作を慣らします。

WindowsでPowerShellを使う場合は ./docker-compose.ps1 のようにパスを調整してください。

トラブルシューティング

  • ポート競合 → .env の APP_PORT, UI_PORT を変更。
  • 権限エラー → Linuxでは sudo chown -R $USER:$USER certs pgdata。
  • HTTPS証明書 → certs/server.crt と server.key を社内CAに差し替え。

次のステップ

セットアップが完了したら、Flows→Planner→Checkerの順に触ってみると操作体系が理解しやすいです。APIページのコピーコマンドも合わせて参照してください。
